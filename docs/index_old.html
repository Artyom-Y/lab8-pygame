🎮 Moving Squares Game

An interactive learning dashboard analyzing a dynamic pygame game featuring autonomous rectangles with intelligent threat detection, color-based aging, and collision physics. Explore architecture patterns, mathematical concepts, and game design principles through interactive diagrams and code analysis.

Project Overview

🎯 Core Concept

A real-time simulation where colored rectangles move autonomously on screen. Each square has a lifecycle: spawn, move, age (color shift), detect threats, flee or chase, then respawn with audio feedback.

🎮 Game Mechanics

Movement: Each rect moves via normalized 2D vector with speed proportional to size. Bounces off walls. Behavior: Larger rects chase smaller ones; smaller rects flee from larger. Lifecycle: Age from blue to red, then respawn.

⚙️ Technical Stack

Core: pygame for rendering and event loop. Data: Config dataclass for constants. Audio: pygame mixer for sound effects. Math: Vector2 normalization, linear interpolation (lerp) for colors, distance calculations.

📊 Key Learning Areas

OOP (MovingRect extends Rect), game loops, vector mathematics, color interpolation, spatial algorithms, collision detection, state management, and pygame fundamentals (Surface, Clock, event handling, drawing).

🔧 Configuration

Screen: 800×800 @ 60 FPS. Squares: 5–30px size, 10 concurrent. Speed: Scaled by inverse width (smaller = faster). Lifespan: 7–25 seconds with color aging.

💡 Design Patterns

Game Loop: Continuous init → events → update → draw → repeat. State Pattern: Each rect encapsulates position, velocity, lifespan. Strategy: Threat detection decoupled from movement logic.

Code Patterns & Analysis

Good Patterns ✅

Configuration Dataclass
Basics
GameConfig encapsulates all tunable parameters (width, fps, size limits, square count) in one immutable structure. Clean, no magic numbers in code.
Engineering Insight
Dataclasses auto-generate __init__, __repr__, __eq__. Makes testing easier: swap CONFIG for mock object. Python standard since 3.7.
Architecture Insight
Single Responsibility: configuration logic separated from game logic. Follow DRY: one place to change screen size, FPS, or entity count.
Object-Oriented Design (MovingRect)
Basics
MovingRect extends pygame.rect.Rect, inheriting position/size logic while adding movement (vector, speed). Methods encapsulate behavior: move_dir, randomize_dir, lerp_color.
Engineering Insight
Inheritance reuses Rect's collision-detection and positional methods. @classmethod random_square() is a factory pattern for convenient object creation with random initialization.
Architecture Insight
Entity pattern common in game engines. Each rect is self-contained, responsible for own state (position, velocity, lifespan, color). Scales naturally to hundreds of entities.
Pure Functions for Utility
Basics
lerp_color, find_threat, escape_threat_vector are stateless functions. No global side effects. Input → computation → output. Easy to test in isolation.
Engineering Insight
Predictable behavior: same inputs always produce same outputs. Facilitates unit testing and debugging. Composition: combine simple functions to solve complex problems.
Architecture Insight
Functional programming principles reduce coupling. Threat logic, movement, color changes are orthogonal concerns, each solvable independently.
Comprehensive Type Hints
Basics
All function parameters and return types annotated. Example: escape_threat_vector(rect: MovingRect, threat: MovingRect, k: int) → pygame.Vector2. Tells IDE and reader intent.
Engineering Insight
Type hints enable static analysis (mypy) to catch errors before runtime. Improves IDE autocomplete and documentation. Best practice in modern Python.
Architecture Insight
Contract clarity: function signature is a contract. Caller knows exactly what to pass; function knows what it receives. Reduces surprises.
Color Interpolation (Lerp)
Basics
lerp_color() blends two colors via linear interpolation: new_color = (1-t)*c1 + t*c2. Parameter t (0 to 1) controls blend: 0=c1, 1=c2, 0.5=midpoint.
Engineering Insight
Lerp is a fundamental utility in graphics/animation. Zips two sequences (RGB components) and interpolates channel-by-channel. Elegant use of zip + list comprehension.
Architecture Insight
Visual feedback (blue→red aging) makes behavior observable to player. Reinforces temporal progression without text. Humanizes the simulation.

Potential Issues ⚠️

Global State Management
Basics
SCREEN, CLOCK, IS_OPEN, FONT, REBIRTH_SOUND, START_COLOR, END_COLOR declared globally, modified in init_window(). Functions depend on implicit global context.
Engineering Insight
Tight coupling: hard to mock in tests. Concurrent calls problematic. Global mutable state = hidden dependencies and action-at-a-distance errors. Difficult to parallelize or reuse logic.
Architecture Insight
Refactor: wrap globals in a Game class or pass context as parameter. Pattern: dependency injection. Benefits: testability, composability, clarity.
Inconsistent Distance Calculations
Basics
find_threat() uses squared distance (dist²) to avoid sqrt(). But escape_threat_vector() computes actual distance (dist). In escape_threat_vector, dist² is computed again unnecessarily.
Engineering Insight
Redundant sqrt() calls. O(n²) per frame because threat detection iterates all rects for each rect. Squared distance is faster for comparisons, but then sqrt needed for magnitude calculations.
Architecture Insight
Opportunity: cache threat results or use spatial partitioning (grid/quadtree). Or precompute squared distances once per frame and reuse.

🔍 Gaps & Opportunities

Global Variables Untyped

SCREEN, CLOCK, FONT, REBIRTH_SOUND declared as None at module level, no type hints. Should annotate: SCREEN: pygame.Surface | None. Improves code clarity and IDE support.

Dataclass Type Hints Optional

GameConfig already has defaults, so @dataclass inference works. But explicit types in @dataclass fields (width: int = 800) is already present and good. Minor: consider frozen=True for immutability.

Callable Types in Callbacks

Event handlers and callbacks not typed. Example: handle_events → None is correct, but no Callable[[Event], None] type hints for event patterns. Low priority for this project.

Performance Analysis

🚀 Wins

Squared Distance Optimization

find_threat() compares sq_distance instead of computing sqrt(). Avoids expensive square-root operation. Floating-point optimization: valid when comparing magnitudes (ordering preserved).

Efficient Vector Normalization

pygame.Vector2.normalize() is optimized C code. Used for direction initialization. Avoids manual normalization: manual_norm = v / len(v) would be slower and less readable.

🐢 Risks

O(n²) Threat Detection

find_threat() iterates all rects for each rect. With 10 rects, 100 distance calculations per frame. At 60 FPS, manageable. But scales poorly: 100 rects = 600,000 distances/sec. Mitigation: spatial indexing (quadtree) or fixed threat pool.

Redundant Distance Computation

Squared distance computed twice: in find_threat() and again in escape_threat_vector() (as dist²). Cache threat result or pass distance to avoid recalculation. Small fix, big payoff in larger games.

3 / 6
Performance Signals: Identified optimization opportunity (threat caching) and scalability risk (O(n²) loop). Current 10-entity cap masked complexity. Good use of vector ops and squared distances where it counts.

💪 Suggested Improvements

Refactor: Dependency Injection

Replace globals with a Game class. Pass GameState to functions. Enables testing, concurrency, and easier debugging. Minimal refactor: wrap SCREEN, CLOCK, etc. in a State object.

Optimize: Cache Threats

Store threat + distance in each rect. Recompute once per frame instead of O(n²) lookups. Or use spatial partitioning (grid) to reduce candidate set.

Add: Particle Effects

Respawn visual: draw fading particles at old position. Or sprite animations. Reinforces visual feedback. Minimal code: yield particles from respawn, draw then remove.

Extend: Predator/Prey Simulation

Classify rects as predators or prey. Complex behaviors: hunting, herding, starvation. Graph population over time. Emergent complexity from simple rules (Conway's Game of Life vibe).
